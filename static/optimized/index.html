Of course this headline is a completely clickbait thing, but also a real tweet from a Twitter account parody, check out the tweet here.

This page doesn't talk about which framework you should use. It's here to show front end developers how to load web fonts without blocking content from their users, specially to those with a really bad Wi-Fi or mobile connection.

What you should do

  • In your CSS, use a fallback font family by default and swap later to your web font.
  • Use fontfaceobserver by Bram Stein to do the swap when the font is ready.
  • Host your own fonts and use font-display when supported.
  • Use glyphhanger by Filament Group to create subset font files containing only the unicode range you need for your site.
  • Use preload to minimize the FOUT.
  • Use .woff2 format, it has better compression.
